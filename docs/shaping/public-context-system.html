Exocortex shape Rendered planning artifact

Public context engine, private consumers.

Exocortex should be the reusable evidence kernel that Daybook consumes, not a public copy of Daybook. The chosen design is a Rust-first core for source registry, evidence, retrieval profiles, citations, and context packets, projected through CLI, API, MCP, SDK, and a skill.

Work contract

Deliverable: root VISION.md plus a shaping packet, no heavy scaffold and no remote.

Proof now: files exist, HTML opened, boundary language present, and git shows only the intended docs.

Stop if: Daybook private content starts leaking into public defaults, a code scaffold becomes necessary, or a remote/publish step appears.

Standards

Evidence-first, cited, privacy-aware, source-extensible, and aligned with the Factory one-core-many-faces pattern.

Acceptance

A future implementer can tell what belongs in public Exocortex, what stays private in Daybook, and what v0 proves.

Verification

Structural doc checks now. Future fixture: CLI and MCP produce the same packet with fetchable citations and no private-source leakage.

sources -> adapters -> evidence log -> retrieval policy -> context packet
                                      -> CLI / API / MCP / SDK / skill

Public core

Source registry, adapter traits, evidence chunks, citation refs, retrieval profiles, evidence logs, packet builder, fixture-backed privacy/freshness tests, and agent-facing surfaces.

Private consumer

Daybook owns its vault, QMD collection, Monologue paths, clippings, people/project notes, voice, writeback policy, and personal workflows.
