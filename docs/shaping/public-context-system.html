Exocortex shape Docs-only PR contract

Thin public core. Private command centers.

Exocortex is the self-hosted context core that beats ad hoc Markdown without turning into a chat app, vector database product, or public copy of Daybook. One core owns ingest, indexes, evidence, profiles, packets, and citations. CLI, MCP, API, SDK, skills, and any UI are thin projections.

Work contract

Deliverable: update VISION.md, this shaping packet, and the rendered HTML plan.

Proof now: docs cover exo init/ingest/search/packet, MCP tools, public/private boundary, and Daybook migration.

Stop condition: pause after PR. No crates, runtime config, hidden Daybook data, or full build scaffold.

Chosen design

Thin product surface, batteries-included local substrate: source registry, ingest, embedded full-text/vector search, local embeddings, QMD adapter boundary, evidence logs, and packets.

Why it wins

It is easier and safer than hand-rolled Markdown conventions while staying small enough to embed in Daybook, Adminifi, or any future repo.

v0 command surface

01

exo init

Create repo-local config and state. No hosted dependency required.

02

exo ingest

Index files, Markdown, URLs, stdin, transcript/event JSONL, and optional command-backed sources.

03

exo search

Query configured sources by profile and return cited evidence chunks with freshness and trust labels.

04

exo packet

Build compact agent-ready context packets with citations, decisions, gaps, and evidence receipts.

API and SDK

Mirror the core verbs instead of creating a second product: init_workspace, list_sources, ingest_source, search_context, build_context_packet, fetch_source, explain_packet, and suggest_organization.

Daybook migration path

  1. Preserve Daybook as private; no data move and no public config.
  2. Add Daybook-local .exocortex/config.yaml for QMD, vault roots, traces, Monologue, clippings, MOCs, and evidence destination.
  3. Replace selected context-gathering calls with exo packet --profile personal_context.
  4. Write receipts under Daybook-owned evidence or trace paths.
  5. Normalize firehoses through transcript/event JSONL ingest.
  6. Register MCP only after CLI parity and fetchable citations.
  7. Keep Daybook skills private; ship generic Exocortex skills publicly.
  8. Test Adminifi-style overlays after the first read-only migration.
